üìñ Overview

Welcome to the Gratex API documentation. This API provides a complete solution for managing users, clients, quotations (cotizaciones), and invoices (facturas) with token-based authentication.

Base URL: http://your-domain.com/api

Quick Start

1 Register or Login

Create an account or login to get your API token.

POST /api/auth/register
POST /api/auth/login

2 Get Your Token

After login, you'll receive a token in the response.

{
  "success": true,
  "data": {
    "token": "your-api-token",
    "user": {...}
  }
}

3 Use the Token

Include the token in all subsequent requests.

Header: X-API-KEY: your-token
// or
Header: Authorization: Bearer your-token

üîê Authentication

The API uses token-based authentication. Tokens are generated upon login and must be included in the header of all protected requests.

Note: Tokens are stored as SHA256 hashes for security. Keep your token secure and don't share it.
POST /api/auth/register No Auth Required Register a new user account
Request Body 
{
  "email": "user@example.com",
  "password": "yourpassword",
  "name": "John Doe",
  "username": "johndoe"
}
ParameterTypeDescription
email *stringValid email address
password *stringMinimum 4 characters
name *stringFull name (will be split into first/last)
username *stringUnique username (min 3 chars)
Success Response (200) 
{
  "success": true,
  "data": {
    "id": 1,
    "email": "user@example.com",
    "username": "johndoe",
    "name": "John Doe",
    "role": "user"
  },
  "message": "User registered successfully"
}
POST /api/auth/login No Auth Required Login and get access token
Request Body 
{
  "emailOrUsername": "user@example.com",
  "password": "yourpassword"
}
Success Response (200) 
{
  "success": true,
  "data": {
    "token": "32-character-api-token",
    "user": {
      "id": 1,
      "email": "user@example.com",
      "username": "johndoe",
      "name": "John Doe",
      "role": "user"
    }
  }
}
POST /api/auth/signout üîí Auth Required Revoke current token (logout)

No request body required. Include your token in the header.

Success Response (200) 
{
  "success": true,
  "message": "Signed out successfully"
}
GET /api/auth?user_id={id} No Auth Required List all tokens for a user
ParameterTypeDescription
user_id *integerUser ID
Success Response (200) 
["success", [
  {
    "id": 1,
    "created_at": "2024-03-01 10:30:00",
    "last_used": "2024-03-01 11:45:00",
    "is_active": 1
  }
]]
DELETE /api/auth No Auth Required Revoke a specific token by ID
Request Body 
{
  "token_id": 1
}
Success Response (200) 
["success", "Token revoked"]

Same body as creating a cotizacion. Returns base64 PDF content.
